git pull origin main

## Залить изменения в репозиторий

a.  Проверьте имена ваших удаленных репозиториев:
    *   Откройте терминал в вашем локальном репозитории.
    *   Выполните команду:
    git remote -v

b. Подготовка коммита
2.  Проверьте статус (необязательно, но полезно):
    
    git status
    
    Эта команда покажет вам, какие файлы были изменены или добавлены. Вы должны увидеть ваш новый файл в разделе "Untracked files" (Неотслеживаемые файлы).

3.  Добавьте новый файл в "staging area" (область подготовленных файлов):
    *   Для конкретного файла:
        
        git add <имя_вашего_нового_файла>
        
        Например: git add my-new-document.md
    *   Для всех новых/измененных файлов:
        
        git add .
        
        Этот вариант добавляет все неотслеживаемые и измененные файлы в текущей директории и поддиректориях.

4.  Сделайте коммит (создайте снимок состояния репозитория):
    
    git commit -m "Ваше сообщение коммита"
    
    *   -m означает "message" (сообщение).
    *   "Ваше сообщение коммита" должно быть кратким и описательным, например: "docs: Добавлен новый учебный материал по продакт-менеджменту" или "feat: Создан файл для хранения резюме статьи".

5.  Загрузите (Push) коммит на GitHub:
    
    git push origin <ваша_ветка>
    
    *   origin — это стандартное имя для вашего удаленного репозитория на GitHub.
    *   <ваша_ветка> — это название ветки, в которую вы вносите изменения. Чаще всего это main или master. Если вы не уверены, проверьте текущую ветку командой git branch (активная ветка будет отмечена звездочкой).

## Подключиться к репозиторию и скачать проект
1. Установить git: https://git-scm.com/install/windows
2. Поменять директорию через терминал на ту, в которую планируется поместить проект. Пример компнды: D:\Личное\GITHUB\AKnutarev.github.io
3. Клонировать репозиторий через команду в терминале: 
git clone https://github.com/AKnutarev/AKnutarev.github.io.git
4. Провериь подключение к репозиторию командой в терминале:
git remote -v
